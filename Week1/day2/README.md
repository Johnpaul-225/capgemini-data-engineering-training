#  Day 2 – SQL Data Analysis (GROUP BY & JOIN)

##  Objective

This phase is about practicing SQL queries using multiple tables.
We learn how to group data, join tables, and calculate useful insights like totals and averages.

---

##  Problem Statement (Summary)

* Clean the data if needed (handle null values)
* Use GROUP BY to summarize data
* Join Employee and Sales tables
* Calculate metrics like total salary and total sales
* Filter data using WHERE and HAVING
* Generate useful reports like top employees and department performance

📄 Detailed problem statement is available in:
`phase1_problem_statement.pdf`

---

##  Dataset Used

* Dataset: Sample Employee & Sales Data
* Source: Practice dataset (manual creation)
* Tables used:

  * Employee
  * Sales

---

##  Approach

1. Created Employee and Sales tables
2. Inserted sample data into tables
3. Checked data for any missing or incorrect values
4. Used GROUP BY to calculate totals, averages, and counts
5. Applied JOINs to combine Employee and Sales data
6. Filtered results using WHERE and HAVING
7. Generated final outputs like top performers and department reports

---

##  Key Transformations

* GROUP BY with SUM, COUNT, AVG
* INNER JOIN and LEFT JOIN
* HAVING for filtering grouped data
* Simple conditions using WHERE
* Aggregation for reports

---

##  Output

* Total salary by department
* Number of employees in each department
* Total sales by employee
* Top employees based on sales
* Department-wise sales performance

---

##  Challenges Faced

* Understanding when to use WHERE vs HAVING
* Writing correct JOIN conditions
* Grouping data properly without errors

---

##  Learnings

* Learned how GROUP BY works in SQL
* Understood different types of JOINs
* Learned how to calculate business metrics
* Gained confidence in writing real-world SQL queries
* Understood how to turn raw data into useful insights
