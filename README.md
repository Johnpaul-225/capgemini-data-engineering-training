# Data Engineering Training
##  Objective
Build a complete data engineering pipeline from basic SQL queries to advanced PySpark transformations and real-world analytics using a multi-table dataset.

This project demonstrates the journey from fundamentals to building a full business reporting pipeline.

---

##  Problem Summary

- Learn SQL basics and convert them into PySpark  
- Perform data cleaning on messy datasets  
- Build ETL pipelines (Extract → Transform → Load)  
- Work with multiple datasets and joins  
- Apply aggregations and business logic  
- Implement segmentation techniques  
- Use window functions for advanced analytics  
- Build a final reporting dataset  

---

##  Project Phases

### 🔹Phase 0 – Databricks Learning Pathway
Completed foundational modules on:

- Data ingestion (Lakeflow Connect)
- Workflow orchestration (Lakeflow Jobs)
- Pipeline development (Spark Declarative Pipelines)
- DevOps fundamentals
- Includes certification badges and summary documentation


### 🔹 Phase 1 – SQL to PySpark Foundations  
Learn basic SQL queries and their PySpark equivalents.  
Focus on filtering, selecting, and grouping data.

###  🔹 Phase 2 – Data Transformation & Analysis
Working with real datasets
Data cleaning and validation
Joins, aggregations, and business queries
Customer and sales data analysis

--- 

### 🔹 Phase 3A – Data Cleaning  
Work with messy data:
- Handle null values  
- Remove duplicates  
- Filter invalid data  

Ensures data quality before analysis.

---

### 🔹 Phase 3 – ETL Pipeline  
Build a basic ETL pipeline:
- Read datasets  
- Clean and join data  
- Calculate metrics like total spend  
- Generate initial business insights  

---

### 🔹 Phase 4 – Business Pipeline  
Create business-level analytics:
- Daily sales  
- City-wise revenue  
- Top customers  
- Repeat customers  
- Final reporting table  

---

### 🔹 Phase 4A – Segmentation & Bucketing  
Divide customers into categories:
- Gold / Silver / Bronze  
- Use multiple methods:
  - Conditional logic  
  - Quantile-based  
  - Bucketizer  
  - Window functions  

---

### 🔹 Phase 5 – Databricks + Olist (Real Dataset)  
Work with real-world data:
- Join multiple tables  
- Use window functions  
- Perform advanced analytics  
- Build a complete reporting pipeline  

---

##  Approach

- Loaded datasets into PySpark DataFrames  
- Cleaned and validated data  
- Joined multiple tables  
- Applied aggregations and filters  
- Used window functions for ranking and trends  
- Implemented segmentation logic  
- Built final reporting datasets  

---

##  Key Transformations

- Data cleaning (`dropna`, duplicates removal)  
- Filtering invalid data  
- Joins (inner, left, multi-table joins)  
- Aggregations (`sum`, `avg`, `count`)  
- Window functions (`row_number`, `dense_rank`, running total)  
- Conditional logic (segmentation)
